Detailed Description

Parse in PQL query and build query tree.

Used to parse in PQL query

Workflow:

  1. parse the query
  2. build query tree in table format
  3. validate the query
  4. make optimisation for fast performance
  5. set dependancy for query variables

Find the index of given string in queryVarTable.

Parameters
strstring
flaginteger Use of flag
Reduce the loads of traverse and do validation
0 The variable don't need to exist 1 The variable must exist 2 Type boolean 3 The variable will be a variable 4 The variable will be a procedure 5 '_' 6 stmt_num

Process:

  1. get the index of variables found inside the query body
  2. if new special variables will be added
  3. if it's already there then return the index
    else return -1

    Cases:
  1. '_' ->underscore
  2. boolean ->boolean
  3. integer ->stmt num
  4. "NAME" ->known

    Validates:
  1. If known variables exist in pkb
  2. If the required QueryVariable type is stmt
    Note
    The flag 1 is a trick of playing with avaliability, thus variables variable and assign cannot be tested in this function
    Returns
    int the index in queryVarTable or -1

bool QueryPreprocessor::setupVarTable ( vector< string >  declares)
private

insert the declared variables into the var table

Parameters
declaresvector<string> elements in this vector is alraedy well formed

example: if a, b stmt c, d

Process:

  1. Loop through the declare table and insert value to QueryVarTable
  2. Validate each variable

    Cases:
  1. Only one variable is declared
  2. More than one variable is declared

    Validates:
  1. Name should be synonms
  2. Variable cannot be duplicate
    Note
    In order to check the duplicate, all flags for getIndexFromVarTable should be set to 0, which indicate no adding is allowed.
    See Also
    Type
    QueryVariable
    Returns
    The complitance of this function, true for done, false for error
